# Test Cases & Brief

This file contains concise test case descriptions and run instructions for key specs in this workspace.

## 1) Facebook Login
- Purpose: Verify a user can log in to Facebook with valid credentials.
- Precondition: Network access and valid test credentials.
- Steps:
  1. Navigate to `https://www.facebook.com`.
  2. Fill `input[name="email"]` with the username/email.
  3. Fill `input[name="pass"]` with the password.
  4. Click `button[name="login"]`.
  5. Wait for the page to stabilize (use `networkidle` or a specific element selector).
- Verification (preferred order):
  - Check a logged-in element exists (profile avatar, account dropdown, or new feed container).
  - Fallback: check URL or presence of user-specific UI; avoid relying only on the page title.
- Run command: `npx playwright test tests/Facebook.spec.js --headed`

## 2) E2E Order Flow (rahulshetty)
- Purpose: End-to-end order placement for a product using data-driven inputs from `Utils/dataTest.json`.
- Precondition: App is accessible and test data file contains valid entries.
- Steps:
  1. Open application login page (login page object `loginPage.goto()`).
  2. Login using credentials from the data set.
  3. On dashboard, add the configured `productname` to cart (`dashboard.addProductToCart(productname)`).
  4. Go to cart (`dashboard.gotoCart()`).
  5. Proceed to checkout (`checkout.gotoCheckout()`).
  6. Select country `India` (`paymentPage.selectCountry("India")`).
  7. Place the order (`paymentPage.placeOrder()`).
- Verification:
  - Assert the thank-you message is visible and matches `Thankyou for the order.` (trim whitespace when comparing).
  - Optionally verify order appears in Order History if supported.
- Run command: `npx playwright test tests/rahulframe.spec.js`

## Notes & Recommendations
- Use element-specific assertions rather than page title checks for robust validation.
- Trim and normalize strings when asserting visible text (e.g., `.trim()` or use `toHaveText(/Thankyou for the order\.?/i)`).
- If tests are flaky, add explicit waits for the element you assert (e.g., `await expect(locator).toBeVisible()` before text check).

## 3) Intrait site availability
- Purpose: Verify the `https://intrait.web.bc/cgi-bin/index` entry responds and the site is reachable.
- Precondition: Network access to the intrait host/domain.
- Steps:
  1. Navigate to `https://intrait.web.bc/cgi-bin/index`.
  2. Wait for network idle or DOM content loaded.
  3. Confirm the navigation returned a successful HTTP status (< 400).
  4. Optionally assert a page-specific element or URL fragment to verify correct page.
- Verification:
  - Assert the navigation response status is less than 400.
  - Optionally check for a stable UI element (e.g., a form or header) present on the page.
- Run command: `npx playwright test tests/intrait.spec.js`
